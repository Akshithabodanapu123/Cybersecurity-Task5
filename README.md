# Task 5 - Capstone Project & Incident Response

## Project Title
Website Security Assessment and Incident Response Plan

## Objective
The objective of this project is to understand common website security risks and prepare an incident response plan to improve web application security.

## Project Planning
- Study common web security threats.
- Identify possible vulnerabilities.
- Prepare an incident response plan.
- Recommend security improvements.

## Implementation
The project explains common website vulnerabilities such as SQL Injection, Cross-Site Scripting (XSS), weak passwords, insecure cookies, and missing security headers. It also describes security best practices to reduce these risks.

## Incident Response Simulation

### Identification
Detect the security incident and identify the affected system.

### Containment
Isolate the affected system to prevent further damage.

### Eradication
Remove the threat and fix the vulnerability.

### Recovery
Restore normal system operations and verify security.

### Lessons Learned
Review the incident and implement preventive measures.

## Conclusion
This project demonstrates the importance of identifying security risks and following an effective incident response process to protect websites from cyber attacks.
